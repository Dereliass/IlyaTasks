Работа с XML


Краткая задача:

Создать класс/пакет, в котором будет реализована валидация XML документов. Язык для выполнения - Java.

Техническое задание:

Класс/пакет должен обладать функциональностью валидации XML файлов.
В случае валидности - выдавать осмысленное сообщение о валидности файла.
В случае невалидности выдавать осмысленное сообщение о невалидности с номером строки в документе и конкретным тэгом в котором допущена ошибка, а так же иметь поле/return значение функции, типа boolean: true - валидный, false - невалидный.


Список проверок для валидации
Проверка Описание Пример
Проверка на закрытость тэгов Каждый тэг дожне быть закрыт, иначе документ не валиден <customized>d= a</customized> - валидно;
<customized>d= a<customized> - не валидно.
Проверка на разрешенные тэги Если в документе присутствуют тэги которые не разрешены явным образом , то документ не валиден <customized>d= a</customized> - валидно;
<dream>d= a</dream> - не валидно.
Проверка на разрешенные атрибуты Некоторые тэги имеют атрибуты, если атрибуты тэга не разрешены явным образом, то документ не валиден
Примечание: аттрибут может отсутствовать, т.е. аттрибуты не являются обязательными		 <rule library="INSTJCL"> - валидно;
<rule foo="bar"> - не валидно.
Проверка на корректность контекста Некоторые тэги могут могут быть использованы только внутри других тэгов, если они находятся вне них, то документ не валиден <rulesList><rule library="INSTJCL"></rule></rulesList> - валидно;
<rule library="INSTJCL"><rulesList></rulesList> - не валидно.

Пояснения:

Валидация - проверка XML файлов на корректность синтаксиса.
Parent элемент - тот тэг внутри которого должен находиться данный тэг.

Условия:

  1. Задание необходимо оформить в виде Pool request к данному репозиторию.
  2. Должен быть соблюден Google code style.
  3. В случае использования сторонних библиотек необходимо обосновать их использование.
  4. Программа должна быть отказоустойчива.
  5. Код должен пройти внутренний code review.

Ограничения:

  1. Запрещено использование сторонних библиотек опубликованных под лицензиями кроме MIT и Apache.
  2. Запрещено использование брутфорс решения (парсить файл, как текстовый). Необходимо обрабатывать файл, как XML документ.
