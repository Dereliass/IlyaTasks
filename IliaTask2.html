Работа с XML


Краткая задача:

Создать класс, в котором будет реализована валидация XML документов.

Техническое задание:

Класс должен обладать функциональность валидации XML файлов.
В случае валидности - выдавать сообщение о валидности файла.
В случае невалидности выдавать осмысленное сообщение о невалидности с номером строки и конкретным тэгом в котором допущена ошибка, а так же иметь поле/return значение функции, типа boolean: true - валидный, false - невалидный.

Пояснения:

Валидация - проверка XML файлов на закрытость тэгов, во избежание SAXException.

Пример:

<customized>d= a<customized> - Не валидно.

<customized>d= a</customized> - Валидно.

Условия:

  1. Задание необходимо оформить в виде Pool request к данному репозиторию.
  2. Должен быть соблюден Google code style.
  3. В случае использования сторонних библиотек необходимо обосновать их использование.
  4. Программа должна быть отказоустойчива.
  5. Код должен пройти внутренний code review.

Ограничения:

  1. Запрещено использование сторонних библиотек опубликованных под лицензиями кроме MIT и Apache.
  2. Запрещено использование брутфорс решения (парсить файл, как текстовый). Необходимо обрабатывать файл, как XML документ.
